# news-report — HTML 뉴스 브리핑 생성

DB에 적재된 정제 기사를 사람이 읽는 브리핑으로 변환하는 마지막 단계.

## 실행

```bash
# 최근 7일, 전체
python .claude/skills/news-report/scripts/build_report.py \
  --days 7 --out reports/news_$(date +%Y-%m-%d).html

# 특정 시점 이후 + 중요 기사만(스코어 30↑)
python .claude/skills/news-report/scripts/build_report.py \
  --since 2026-07-01 --min-score 30 --out reports/weekly.html
```

인자:
- `--days N` 또는 `--since YYYY-MM-DD` (기간 필터, pub_date 기준. 공란 날짜는 항상 포함)
- `--min-score` 중요도 하한(정기 브리핑은 20~30 권장, 아카이브는 0)
- `--out` 출력 경로(필수)

## 리포트 구성

- **헤더**: 기준일·기간·총건수·스코어 하한·생성시각
- **카테고리 칩 바**: 6개 카테고리별 건수 한눈에 (추세 파악)
- **카테고리 섹션**: 색상 코딩(정책 빨강/시장 파랑/경매 주황/재개발 초록/도시계획 보라/산업단지 청록), 각 섹션 내 스코어 내림차순
- **기사 카드**: 스코어 배지 + 제목(원문 링크) + 요약 + 출처·발행일·티어 배지 + 교차출처 배지 + 수집방법(api/web/crawl) + 키워드 태그

## 디자인 표준

지식베이스 경매 보고서 표준을 따른다: 다크 네비 헤더(`#1e293b`), 화이트 콘텐츠, 카테고리 색상 코딩, 카드형 그리드(반응형 `auto-fill minmax(320px)`). 단일 HTML 파일(외부 의존 없음)이라 브라우저·아티팩트·이메일 첨부 어디서든 열린다.

## 아티팩트 발행(선택)

사용자가 공유용 웹페이지를 원하면 생성된 HTML을 Artifact 도구로 발행할 수 있다. 단, 뉴스 요약은 사실 기반 자기 산출물이므로 발행 가능하나, 특정 개인 대상이거나 매체를 사칭하는 형태가 아니어야 한다(단순 뉴스 큐레이션은 문제없음).

## 왜 이렇게 하나

- **스코어 정렬 + 하한**: 정기 수집은 양이 많다. 중요도로 정렬하고 하한을 두어 "읽을 것"만 위로 올린다.
- **교차출처 배지 노출**: 여러 매체가 다룬 사안임을 시각적으로 표시해 독자가 이슈 크기를 즉시 판단한다.
- **원문 링크**: 제목 클릭 시 재배포가 아닌 원문으로 — 신뢰·심층 확인 경로 확보.

## 협업

DB(`data/news.db`)만 있으면 독립 실행된다. curator 적재 완료 후 호출된다. 생성 경로를 오케스트레이터에 반환한다.
